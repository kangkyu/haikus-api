Haikus-API Documentation

ApiKeysController

  • POST /api_keys with valid params and valid authorization headers (success)

HaikusController

  • GET haikus, with a valid token (success)
  • GET haikus/:id with a valid token (success)
  • POST haikus with a valid token and valid params (success)
  • PATCH haikus with a valid token and valid params (success)
  • DELETE /haikus with a valid token and valid id (success)

LinesController

  • GET /lines, with proper authorization headers (success)
  • POST lines, with correct params and correct authorization headers (success)
  • PATCH /lines, with correct params and correct authorization headers (success)
  • GET /lines/:id, with correct authorization headers (success)
  • DELETE /lines/:id, with correct :id and authorization headers (success)

UsersController

  • POST /users, with correct params and auth headers (success)
  • DELETE /users, with correct id and auth headers (success)
